### [SLIDE 3] The Memory Problem (0:10–0:50)

There are two problems nobody's solved well.

The first is platform chaos. If you're a content creator on Douyin, TikTok, and Red Note, each platform gives you a different dashboard speaking a different language. Completion rate here, viral ratio there, save rate somewhere else. You want the full picture? You're stitching spreadsheets at two in the morning.

The second is context loss. You spec eighteen signals on Monday. By Wednesday, your AI agent has forgotten half of them. By Friday, it's reimplementing things that already exist. Context dies between sessions — and when it dies, engineering time burns.

We solved both. Deterministic engineering for the first. EverMemOS for the second.

---

### [SLIDE 4] How We Used EverMemOS (0:50–1:40)

Our setup: Claude Code for deep implementation — writing the statistics library, the signal framework, all eleven analysis engines. OpenClaw for orchestration — scheduling tasks, managing multi-agent workflows, tracking progress across weeks.

Two different AI systems. One shared memory layer through EverMemOS.

Day one, we defined eighteen intelligence signals across five categories. Ten days later, when Claude Code started a fresh session to build the cross-platform comparator, it didn't need to be re-briefed. Episodic memory gave it the full signal definitions. Trait memory gave it the architectural constraints. It picked up exactly where we left off.

When OpenClaw needed to verify that Phase 3 hadn't broken Phase 1's data pipeline, it pulled the original schema decisions from memory — not from a stale document.

No re-explaining. No context drift. No rework.

---

### [SLIDE 5] What We Built (1:40–2:20)

So what did memory-driven development actually produce?

Dash Persona. A creator intelligence engine that unifies Douyin, TikTok, and Red Note into one view.

Persona Score — zero to one hundred across six dimensions. Click any number, see the exact formula.

Cross-platform comparison on a single radar chart with adaptive gap detection.

Content Calendar computed from your own best-performing time slots. Not AI guesswork — math.

And the key word: **deterministic.** No LLM calls. No API keys. Same input, same output, every time.

---

### [SLIDE 6] Memory In Practice (2:20–3:10)

Here's what the timeline looked like.

Claude Code shipped the statistics library, the signal framework, and all eleven engines. Three hundred eight unit tests. Fifteen end-to-end cases. Every score traces back to a readable formula — and none of it needed re-briefing across sessions.

OpenClaw orchestrated multi-agent workflows. When Phase 3 integration landed, it verified against Phase 1's pipeline by pulling schema decisions straight from EverMemOS.

The engineering result: Next.js 16, React 19, Tailwind CSS 4. Eleven analysis modules. Eighteen signals. Seven data adapters. An Electron desktop collector and a Chrome extension — all built with persistent memory across every session.

---

### [SLIDE 7] The Double Memory (3:10–3:50)

There are two layers of memory in this project.

Layer one — EverMemOS gave our AI agents memory, so they could build a complete product in ten days without losing a single engineering detail.

Layer two — Dash Persona itself gives creators memory. Historical snapshots. Growth trends over time. Strategy experiments tracked on a decision tree. All local. All persistent.

The AI's memory built the tool. The tool gives humans memory.

**When you solve memory for machines, you can build things that solve memory for people.**
